# Using simple-blog-ui with React

The library ships official **React wrappers** so every component becomes a regular PascalCase React component with typed props and standard event handlers (`onClick`, `onChange`, etc.). Works with React 17, 18, and 19 (JS or TS).

## Installation

```bash
npm install simple-blog-ui
```

`react` and `react-dom` are declared as optional peer dependencies — you only install them if you're building a React app.

## Setup

Import the CSS once (e.g. in your root `main.jsx` / `_app.tsx`):

```js
import 'simple-blog-ui/style.css';
```

Then import components from the `/react` subpath wherever you need them:

```jsx
import { Button, Input, Alert } from 'simple-blog-ui/react';
```

## Basic example

```jsx
import { Button, Input, Card, Stack } from 'simple-blog-ui/react';
import 'simple-blog-ui/style.css';

export default function LoginForm() {
  const [email, setEmail] = useState('');

  return (
    <Card padding="lg" style={{ maxWidth: 420 }}>
      <span slot="header">Sign in</span>
      <Stack gap="4">
        <Input
          label="Email"
          placeholder="you@example.com"
          onInput={(e) => setEmail(e.detail.value)}
        />
        <Button variant="primary" onClick={() => console.log('login', email)}>
          Continue
        </Button>
      </Stack>
    </Card>
  );
}
```

## Controlled components

The wrappers accept the same props the underlying custom elements expose. For form inputs, keep your state in React and mirror it into the component:

```jsx
const [comment, setComment] = useState('');

<Textarea
  label="Comment"
  value={comment}
  onInput={(e) => setComment(e.detail.value)}
/>
```

## Modal and overlay triggers

Modal/Offcanvas expose imperative `show()` and `close()` methods on the underlying element. Access them via a `ref`:

```jsx
import { useRef } from 'react';
import { Modal, Button } from 'simple-blog-ui/react';

function Example() {
  const modal = useRef(null);
  return (
    <>
      <Button onClick={() => modal.current.show()}>Open modal</Button>
      <Modal ref={modal} title="Confirm">
        <p>Are you sure?</p>
        <Button slot="footer" onClick={() => modal.current.close()}>
          Close
        </Button>
      </Modal>
    </>
  );
}
```

## TypeScript

The PascalCase wrappers accept the same props as the underlying elements. Declaration files ship automatically with the package — no extra setup needed in a TypeScript project.

```tsx
import { Button, ButtonProps } from 'simple-blog-ui/react';

const buttonProps: ButtonProps = {
  variant: 'primary',
  size: 'lg',
  onClick: () => {},
};
```

## Using raw custom elements (fallback)

If you prefer to use the tags directly, it still works:

```jsx
import 'simple-blog-ui';
import 'simple-blog-ui/style.css';

<sb-button variant="primary">Click me</sb-button>
```

In **React 18**, custom events (`sb-click`) need a `ref` + `addEventListener`:

```jsx
const ref = useRef();
useEffect(() => {
  const handler = () => console.log('clicked');
  ref.current?.addEventListener('sb-click', handler);
  return () => ref.current?.removeEventListener('sb-click', handler);
}, []);

<sb-button ref={ref}>Click</sb-button>
```

In **React 19**, custom elements and events are first-class — events can be written inline (`onSb-click={fn}`).

The PascalCase wrappers from `/react` eliminate this divergence across React versions.
