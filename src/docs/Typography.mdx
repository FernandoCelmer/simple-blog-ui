Single family across the whole library: **Fira Mono**, monospace. Same scale as [`mkdocs-simple-blog`](https://github.com/FernandoCelmer/mkdocs-simple-blog)'s `main.css`.

Headings are **uppercase** with `letter-spacing: 0.0625em`. Use `<sb-heading>` for a consistent semantic element across shadow DOM boundaries.

## Body text

- Family: `"Fira Mono", monospace`
- Base size: `1.2em` — remember `html { font-size: 80% }` is applied, so `1rem ≈ 12.8px`
- Line height: `1.8em`
- Letter spacing: `0.0625em`

Use `<sb-text>` for paragraph-level copy with variants (`body`, `muted`, `small`, `italic`).

## Inline and block code

```html
<sb-code>inline code</sb-code>
<sb-code block language="javascript">
const greet = (name) => `Hello, ${name}!`;
</sb-code>
```

`sb-code` uses `--code-bg` and `--code-fg` tokens (which stay dark in both themes — terminal-like). Block variant supports syntax highlighting via [highlight.js](https://highlightjs.org/) — pass a `language` attribute or let auto-detect run.

## Buttons, nav, and badges

All interactive labels follow the same uppercase convention as headings:

- `text-transform: uppercase`
- `letter-spacing: 0.1rem`
- Font weight: 400 (normal) — bold only for badge titles

This gives the whole UI a typewriter / terminal rhythm without relying on weight variation.
