# simple-blog-ui

Framework-agnostic UI component library for **simple-blog**. Built with [Lit](https://lit.dev/) (Web Components). Works with **plain HTML**, **React** (JS or TS), and **Vue**.

## Design language

Inherited from [`mkdocs-simple-blog`](https://github.com/FernandoCelmer/mkdocs-simple-blog):

- Font: **Fira Mono** monospace
- Palette: black + white with accent colors (blue, green, yellow, orange, purple, red)
- Borders: 2px solid black
- Uppercase headings, buttons, and nav labels with `letter-spacing: 0.0625em`
- No rounded corners — deliberate, rigid, document-first aesthetic
- Light and dark themes via `data-theme` attribute

## Installation

```bash
npm install simple-blog-ui
```

## Usage

Pick your stack for a focused integration guide:

- **[Usage / HTML](?path=/docs/usage-html--docs)** — plain HTML, bundler or CDN
- **[Usage / React](?path=/docs/usage-react--docs)** — PascalCase wrappers via `simple-blog-ui/react`
- **[Usage / Vue](?path=/docs/usage-vue--docs)** — Vue 3 with `isCustomElement`

Minimal React example:

```jsx
import { Button, Input } from 'simple-blog-ui/react';
import 'simple-blog-ui/style.css';

<Button variant="primary" onClick={() => {}}>Click me</Button>
<Input label="Email" onInput={(e) => console.log(e.detail.value)} />
```

Minimal Vue example:

```vue
<script setup>
import 'simple-blog-ui';
import 'simple-blog-ui/style.css';
</script>

<template>
  <sb-button variant="primary" @sb-click="onClick">Click me</sb-button>
</template>
```

## Theming

Two themes out of the box — toggle with the **Light / Dark** button in the Storybook toolbar above. In consumer apps, set `data-theme="dark"` on `<html>` to enable dark mode:

```html
<html data-theme="dark"> ... </html>
```

All components read their colors, spacing, and typography from CSS custom properties defined in `root.css`. Override any token on `:root` (or a scoped ancestor) to re-theme:

```css
:root {
  --primary: #1e40af;
  --background: #fef3c7;
  --color-red: #b91c1c;
}
```

See **Design / Colors** and **Design / Typography** for the full token map.
